Reglas fundamentales — siempre aplicar

Option Explicit
Primera línea de CADA módulo. Obliga a declarar todas las variables. Previene el 80% de los bugs más comunes.
Tipado explícito
Nunca usar Dim x sin tipo. Siempre Dim x As String. Variant solo cuando sea estrictamente necesario.
Sin Select/Activate
Nunca usar Range.Select ni Sheet.Activate. Trabajar directo con las referencias al objeto.
Error handling
Todo Sub/Function con lógica de negocio debe tener On Error GoTo Handler. Nunca On Error Resume Next sin motivo.

Documentación — Convención BPS

VBA no tiene docstrings nativos. La convención del equipo usa un bloque de comentarios estructurado antes de cada Sub/Function:

'==============================================================================
' Módulo:   ModuloComprobantes
' Propósito: Procesamiento de comprobantes fiscales para reportes TP
' Autor:     Dev Team BPS
' Creado:    2024-01-15
' Modificado:2025-02-10
'==============================================================================
Option Explicit

'------------------------------------------------------------------------------
' Función:  ValidarCUIT
' Resumen:  Verifica que un CUIT tenga formato válido (XX-XXXXXXXX-X).
' Params:
'   cuit (String): CUIT a validar, puede incluir guiones o no.
' Retorna:
'   Boolean: True si el CUIT es válido, False en caso contrario.
' Ejemplo:
'   ValidarCUIT("20-12345678-5") → True
'   ValidarCUIT("12345")         → False
'------------------------------------------------------------------------------
Public Function ValidarCUIT(cuit As String) As Boolean
    Dim soloNums As String
    soloNums = Replace(Replace(cuit, "-", ""), " ", "")
    ValidarCUIT = (Len(soloNums) = 11 And IsNumeric(soloNums))
End Function

Manejo de errores — patrón estándar

Public Function ProcesarComprobante(comp As TComprobante) As Boolean
    On Error GoTo ErrorHandler

    ' ── Validaciones ──────────────────────────────────
    If Not ValidarCUIT(comp.CUIT) Then
        Err.Raise vbObjectError + 1001, "ProcesarComprobante", _
                  "CUIT inválido: " & comp.CUIT
    End If

    ' ── Lógica principal ──────────────────────────────
    ProcesarComprobante = True
    Exit Function

ErrorHandler:
    LogError "ProcesarComprobante", Err.Number, Err.Description
    ProcesarComprobante = False
End Function

'------------------------------------------------------------------------------
' Centralizar el logging de errores en un solo lugar
'------------------------------------------------------------------------------
Private Sub LogError(source As String, errNum As Long, errDesc As String)
    Dim msg As String
    msg = Format(Now, "yyyy-mm-dd hh:nn:ss") & _
          " | ERROR | " & source & " | #" & errNum & ": " & errDesc
    Debug.Print msg
    ' También escribir a una hoja de log si existe
    On Error Resume Next
    Sheets("_LOG").Cells(Rows.Count, 1).End(xlUp).Offset(1).Value = msg
    On Error GoTo 0
End Sub

Trabajar con rangos — sin Select

' ❌ MAL — usa Select, lento y frágil
Sub MalEjemplo()
    Sheets("Datos").Activate
    Range("A1").Select
    Selection.Value = "Hola"
End Sub

' ✅ BIEN — referencia directa, sin activar nada
Sub BuenEjemplo()
    Dim ws  As Worksheet
    Dim rng As Range

    Set ws  = ThisWorkbook.Sheets("Datos")
    Set rng = ws.Range("A1")

    rng.Value = "Hola"
    Set rng = Nothing
End Sub

' Leer una tabla completa en array (mucho más rápido que celda por celda)
Sub LeerTablaEnArray()
    Dim ws    As Worksheet
    Dim datos() As Variant
    Dim i     As Long

    Set ws = ThisWorkbook.Sheets("Datos")

    ' Una sola lectura de memoria — 100x más rápido que Loop
    datos = ws.Range("A2:D" & ws.Cells(ws.Rows.Count, 1).End(xlUp).Row).Value

    For i = 1 To UBound(datos, 1)
        Debug.Print datos(i, 1), datos(i, 2)
    Next i
End Sub

Optimización — acelerar macros

' Siempre encapsular operaciones grandes con esto
Sub OptimizarInicio()
    Application.ScreenUpdating   = False
    Application.Calculation      = xlCalculationManual
    Application.EnableEvents     = False
    Application.DisplayAlerts    = False
End Sub

Sub OptimizarFin()
    Application.ScreenUpdating   = True
    Application.Calculation      = xlCalculationAutomatic
    Application.EnableEvents     = True
    Application.DisplayAlerts    = True
End Sub

' Uso — siempre con manejo de error para restaurar el estado
Sub MiMacroGrande()
    On Error GoTo Cleanup
    OptimizarInicio

    ' ... tu código acá ...

Cleanup:
    OptimizarFin
    If Err.Number <> 0 Then MsgBox Err.Description, vbCritical
End Sub
⚠️ Si OptimizarFin no se llama (error sin handler), Excel queda en modo manual de cálculo y el usuario no entiende por qué no funcionan las fórmulas. Siempre restaurar en un bloque Cleanup.
