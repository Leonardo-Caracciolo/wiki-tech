📊 Testing VBA — Rubberduck

Unit testing en VBA con Rubberduck, estrategias de testing sin add-ins y patrones de diseño testeable para macros Excel.

¿Por qué testear VBA?

El código VBA sin tests es el más frágil del stack del equipo. Un cambio en la estructura de un Excel puede romper silenciosamente una macro que llevaba meses funcionando. Los tests detectan esas roturas antes que el usuario.

Rubberduck — Unit Testing integrado

Rubberduck es un add-in gratuito para VBE que agrega capacidades de unit testing similares a NUnit/pytest directamente en el IDE.

Instalación

' 1. Descargar de: https://rubberduckvba.com/
' 2. Cerrar Excel
' 3. Ejecutar el instalador
' 4. Abrir Excel → Rubberduck aparece en la barra de herramientas
' 5. Rubberduck → Unit Tests → Test Explorer

Tests con Rubberduck — Template base

'@TestModule
'@Folder("Tests")
Option Explicit
Option Private Module

'@TestMethod("ValidarCUIT")
Public Sub ValidarCUIT_CUITValido_RetornaTrue()
    ' Arrange
    Dim cuit As String
    cuit = "20123456789"

    ' Act
    Dim resultado As Boolean
    resultado = ValidarCUIT(cuit)

    ' Assert
    Assert.IsTrue resultado, "CUIT válido de 11 dígitos debe retornar True"
End Sub

'@TestMethod("ValidarCUIT")
Public Sub ValidarCUIT_CUITCorto_RetornaFalse()
    ' Arrange
    Dim cuit As String
    cuit = "123"

    ' Act
    Dim resultado As Boolean
    resultado = ValidarCUIT(cuit)

    ' Assert
    Assert.IsFalse resultado, "CUIT de menos de 11 dígitos debe retornar False"
End Sub

'@TestMethod("ValidarCUIT")
Public Sub ValidarCUIT_StringVacio_RetornaFalse()
    Assert.IsFalse ValidarCUIT(""), "String vacío debe retornar False"
End Sub

Test setup y teardown

'@TestModule
Option Explicit
Option Private Module

Private wsTest As Worksheet  ' Hoja temporal para los tests

'@ModuleInitialize
Public Sub ModuleInitialize()
    ' Se ejecuta UNA VEZ al inicio del módulo de tests
    Application.ScreenUpdating = False
End Sub

'@ModuleCleanup
Public Sub ModuleCleanup()
    ' Se ejecuta UNA VEZ al final del módulo
    Application.ScreenUpdating = True
End Sub

'@TestInitialize
Public Sub TestInitialize()
    ' Se ejecuta ANTES DE CADA test — setup limpio
    Set wsTest = ThisWorkbook.Worksheets.Add()
    wsTest.Name = "_TestTemp"
End Sub

'@TestCleanup
Public Sub TestCleanup()
    ' Se ejecuta DESPUÉS DE CADA test — cleanup
    Application.DisplayAlerts = False
    wsTest.Delete
    Application.DisplayAlerts = True
    Set wsTest = Nothing
End Sub

'@TestMethod
Public Sub ProcesarFila_DatosValidos_EscribeResultado()
    ' Arrange — usar wsTest del TestInitialize
    wsTest.Range("A1").Value = "20123456789"
    wsTest.Range("B1").Value = 12100

    ' Act
    ProcesarFila wsTest, 1

    ' Assert
    Assert.AreEqual "OK", wsTest.Range("C1").Value
End Sub

Testing sin Rubberduck — Patrón manual

Cuando no se puede instalar Rubberduck (restricciones corporativas), usar este patrón de testing manual:

'==============================================================
' Módulo de tests manuales — ejecutar RunAllTests() para correr
'==============================================================
Option Explicit

Private passed As Long
Private failed As Long

'------ Helpers de assertion ------

Private Sub AssertTrue(condition As Boolean, testName As String)
    If condition Then
        passed = passed + 1
        Debug.Print "  ✓ " & testName
    Else
        failed = failed + 1
        Debug.Print "  ✗ FALLO: " & testName
    End If
End Sub

Private Sub AssertEqual(expected As Variant, actual As Variant, testName As String)
    If expected = actual Then
        passed = passed + 1
        Debug.Print "  ✓ " & testName
    Else
        failed = failed + 1
        Debug.Print "  ✗ FALLO: " & testName & _
                    " — Esperado: [" & expected & "] Actual: [" & actual & "]"
    End If
End Sub

'------ Suite de tests ------

Public Sub RunAllTests()
    passed = 0: failed = 0
    Debug.Print "=== Tests: ValidarCUIT ==="
    Test_ValidarCUIT_CUITValido
    Test_ValidarCUIT_CUITCorto
    Test_ValidarCUIT_StringVacio
    Debug.Print "Resultado: " & passed & " OK / " & failed & " Fallidos"
End Sub

Private Sub Test_ValidarCUIT_CUITValido()
    AssertTrue ValidarCUIT("20123456789"), "CUIT de 11 dígitos → True"
End Sub

Private Sub Test_ValidarCUIT_CUITCorto()
    AssertTrue Not ValidarCUIT("123"), "CUIT corto → False"
End Sub

Private Sub Test_ValidarCUIT_StringVacio()
    AssertTrue Not ValidarCUIT(""), "String vacío → False"
End Sub

Código testeable — principios de diseño

' ❌ NO testeable — lógica mezclada con Excel
Sub ProcesarHoja()
    Dim i As Long
    For i = 2 To Sheets("Datos").Cells(Rows.Count, 1).End(xlUp).Row
        If Len(Sheets("Datos").Cells(i, 1).Value) = 11 Then
            Sheets("Datos").Cells(i, 3).Value = "OK"
        End If
    Next i
End Sub

' ✅ SÍ testeable — lógica separada de la UI de Excel

' Función pura — fácil de testear
Public Function EsCUITValido(cuit As String) As Boolean
    EsCUITValido = Len(Replace(cuit, "-", "")) = 11 And IsNumeric(Replace(cuit, "-", ""))
End Function

' Orquestador — acepta hoja como parámetro (inyección de dependencia)
Public Sub ProcesarHojaV2(ws As Worksheet)
    Dim i As Long
    Dim lastRow As Long
    lastRow = ws.Cells(ws.Rows.Count, 1).End(xlUp).Row
    For i = 2 To lastRow
        ws.Cells(i, 3).Value = IIf(EsCUITValido(ws.Cells(i, 1).Value), "OK", "ERROR")
    Next i
End Sub
💡 La clave para código VBA testeable: separar la lógica de negocio (funciones puras) de la interacción con Excel (UI). Las funciones puras no tocan Sheets, Cells ni Range — solo reciben y retornan valores.
