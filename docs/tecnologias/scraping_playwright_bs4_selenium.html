01

Playwright

async · chromium · firefox · webkit
Playwright es el estándar moderno para scraping y testing E2E. Controla Chromium, Firefox y WebKit desde Python. Soporta JavaScript nativo, tiene una API limpia y es significativamente más rápido que Selenium. Ideal para SPAs (React, Vue, Angular) y cualquier página que cargue datos via JavaScript.
pip install playwright playwright install ← instala los browsers (Chromium, Firefox, WebKit)
page.goto() + page.title() Abrir una URL y obtener el título de la página
Input
from playwright.sync_api import sync_playwright with sync_playwright() as p: browser = p.chromium.launch(headless=True) page = browser.new_page() page.goto("https://example.com") titulo = page.title() print(titulo) browser.close()
Output
Example Domain
headless=TrueTrue = sin ventana visible. False = abre el browser en pantalla (útil para debug)

wait_until="load" | "domcontentloaded" | "networkidle" — cuándo considerar la página lista

async playwright Versión async para máxima performance (scraping en paralelo)
Input — scraping de múltiples URLs en paralelo
import asyncio from playwright.async_api import async_playwright async def scrape_url(browser, url): page = await browser.new_page() await page.goto(url) titulo = await page.title() await page.close() return titulo async def main(): urls = ["https://site1.com", "https://site2.com", "https://site3.com"] async with async_playwright() as p: browser = await p.chromium.launch() tasks = [scrape_url(browser, url) for url in urls] resultados = await asyncio.gather(*tasks) print(resultados) await browser.close() asyncio.run(main())
Output
["Site 1 - Home", "Site 2 - Bienvenidos", "Site 3 - Dashboard"] # Las 3 páginas se cargaron en paralelo
💡 La versión async permite correr múltiples tabs en simultáneo. Para scraping masivo es la diferencia entre 30 segundos y 5 minutos.

Input — capturar la respuesta de una API interna
import json datos_api = [] def capturar_response(response): if "api/productos" in response.url: datos_api.append(response.json()) page.on("response", capturar_response) page.goto("https://tienda.com/catalogo") page.wait_for_load_state("networkidle") print(datos_api)

BeautifulSoup

html · xml · requests
BeautifulSoup parsea HTML y XML estático. No ejecuta JavaScript, así que necesitás obtener el HTML primero (via requests, httpx, o combinado con Playwright). Es la opción más simple y rápida para páginas clásicas.
pip install beautifulsoup4 requests lxml

Input
import requests from bs4 import BeautifulSoup resp = requests.get("https://quotes.toscrape.com") soup = BeautifulSoup(resp.text, "lxml") # o "html.parser" print(soup.title.text)
Output
Quotes to Scrape
"lxml"Parser más rápido y leniente. Requiere: pip install lxml
"html.parser"Built-in de Python. Más lento pero sin dependencias extra
"html5lib"Más permisivo con HTML mal formado. Más lento
"xml"Para parsear XML en vez de HTML
find() · find_all() Buscar uno o todos los elementos que coincidan
Input
# find() → primer elemento que coincida titulo = soup.find("h1") texto = soup.find("div", class_="quote") precio = soup.find("span", id="precio-total") # find_all() → todos los que coincidan citas = soup.find_all("div", class_="quote") links = soup.find_all("a") # Limitar cantidad primeros = soup.find_all("p", limit=3) # Por múltiples clases elems = soup.find_all("div", class_=["card", "producto"])
Output
# citas es una lista de Tags citas[0].text → "The world as we have created it..." len(citas) → 10
class_=Atributo class (class es palabra reservada en Python, por eso el _)
id=Busca por id HTML
attrs={}Dict de atributos: attrs={"data-id": "123"}
string=Busca por texto exacto o regex: string=re.compile("precio")
limit=Limita la cantidad de resultados
select() · select_one() Buscar con selectores CSS (más potente que find)
Input
# select_one → equivale a find() h1 = soup.select_one("h1") precio = soup.select_one("div.producto > span.precio") # select → equivale a find_all() items = soup.select("ul.lista > li") links = soup.select("a[href^='/productos']") # href que empieza con /productos imgs = soup.select("img[src$='.jpg']") # src que termina en .jpg nth = soup.select("li:nth-of-type(2)") # segundo li
Output
links → [<a href="/productos/remera">, <a href="/productos/pantalon">, ...]
💡 select() usa CSS selectors estándar: >, +, ~, :nth-child(), [attr^=], [attr$=], [attr*=]. Más expresivo que find_all() para estructuras complejas.
.text · .get() · .string · .get_text() Obtener el contenido de un Tag encontrado
Input — dado: <a href="/ruta" class="link">Click aquí</a>
tag = soup.find("a", class_="link") tag.text # → "Click aquí" tag.string # → "Click aquí" (solo si tiene UN solo nodo hijo) tag.get_text() # → "Click aquí" (más seguro, strip=True para limpiar) tag.get("href") # → "/ruta" tag["href"] # → "/ruta" (lanza KeyError si no existe) tag.attrs # → {"href": "/ruta", "class": ["link"]}
Output
get_text(strip=True) → "Click aquí" # sin whitespace extra get("href") → "/ruta" get("id", "N/A") → "N/A" # default si no existe
.textTexto de todos los descendientes concatenado
.stringSolo si el tag tiene exactamente un hijo de texto. Si no, devuelve None
get_text(sep=" ", strip=True)Más control: sep entre bloques, strip limpia whitespace
.get("attr", default)Seguro: devuelve default si el atributo no existe
Navegación del árbol Moverse entre elementos padres, hijos y hermanos
Input
tag = soup.find("li", class_="activo") # Padre tag.parent # → <ul> tag.parent.name # → "ul" # Hijos tag.children # → iterador de hijos directos list(tag.children) tag.find_all(recursive=False) # solo hijos directos # Hermanos tag.next_sibling # puede ser whitespace "\n" tag.next_element # siguiente elemento (más útil) tag.find_next_sibling("li") # siguiente li hermano tag.find_previous_sibling("li")
Ejemplo real — scraping de tabla Extraer datos de una tabla HTML a una lista de dicts

Selenium

webdriver · chrome · firefox
Selenium es el estándar legacy de automatización de browsers. Más lento que Playwright pero con mayor adopción en proyectos legacy y suites de testing enterprise. Desde Selenium 4 no necesita descargar el WebDriver manualmente (usa selenium-manager automáticamente).
pip install selenium ← Selenium 4+ gestiona el driver automáticamente
webdriver.Chrome() — Setup Inicializar el driver y abrir una página
Input
from selenium import webdriver from selenium.webdriver.chrome.options import Options options = Options() options.add_argument("--headless") # sin ventana options.add_argument("--no-sandbox") options.add_argument("--disable-gpu") driver = webdriver.Chrome(options=options) driver.get("https://example.com") print(driver.title) driver.quit() # SIEMPRE cerrar el driver
Output
Example Domain
💡 Usar con context manager para evitar browsers huérfanos: with webdriver.Chrome(options=options) as driver: driver.get("https://example.com")
find_element() · find_elements() Encontrar elementos en la página
Input
from selenium.webdriver.common.by import By # Un elemento h1 = driver.find_element(By.TAG_NAME, "h1") precio= driver.find_element(By.CSS_SELECTOR, ".precio") campo = driver.find_element(By.ID, "email") link = driver.find_element(By.XPATH, "//a[@class='nav-link']") # Múltiples elementos items = driver.find_elements(By.CLASS_NAME, "producto") textos = [el.text for el in items]
By — estrategias de localización
By.IDPor atributo id. El más rápido y específico
By.CSS_SELECTORCSS selector. Recomendado por velocidad y flexibilidad
By.XPATHXPath expressions. Potente pero más lento y frágil
By.CLASS_NAMESolo UNA clase. Para múltiples usar CSS_SELECTOR
By.NAMEPor atributo name (útil para forms)
By.LINK_TEXTTexto exacto de un <a>
By.TAG_NAMEPor nombre de tag HTML
.click() · .send_keys() · .clear() Interaccionar con elementos

WebDriverWait + expected_conditions Esperar explícitamente a que un elemento esté listo
Input
from selenium.webdriver.support.ui import WebDriverWait from selenium.webdriver.support import expected_conditions as EC wait = WebDriverWait(driver, timeout=10) # Esperar que el elemento sea clickeable btn = wait.until(EC.element_to_be_clickable((By.ID, "btn-login"))) btn.click() # Esperar que aparezca en el DOM tabla = wait.until(EC.presence_of_element_located((By.ID, "tabla"))) # Esperar que sea visible modal = wait.until(EC.visibility_of_element_located((By.CLASS_NAME, "modal"))) # Esperar que contenga texto wait.until(EC.text_to_be_present_in_element((By.ID, "status"), "Listo"))
presence_of_element_locatedEl elemento existe en el DOM (puede estar oculto)
visibility_of_element_locatedEl elemento existe Y es visible
element_to_be_clickableEl elemento está visible y habilitado (se puede hacer click)
invisibility_of_element_locatedEl elemento desapareció (spinners, loaders)
⚠️ Nunca usar time.sleep() para esperar. WebDriverWait polling cada 500ms y falla rápido si el elemento aparece. sleep() siempre espera el tiempo completo.

Testing

pytest · unittest · playwright test
El testing de scrapers y automatizaciones tiene dos capas: unit tests (lógica de parsing) y E2E tests (el browser hace lo correcto). Playwright tiene su propio test runner integrado. Para Selenium y BS4 se usa pytest o unittest estándar.
pytest + playwright — E2E testing Testing de flujos completos del browser con fixtures automáticos
Input — test_login.py
# pip install pytest-playwright # playwright install import pytest from playwright.sync_api import Page, expect # El fixture "page" lo provee pytest-playwright automáticamente def test_titulo_correcto(page: Page): page.goto("https://example.com") expect(page).to_have_title("Example Domain") def test_formulario_login(page: Page): page.goto("https://app.com/login") page.fill('input[name="email"]', "[email protected]") page.fill('input[name="password"]', "pass123") page.click("button[type='submit']") # Verificar que redirige al dashboard expect(page).to_have_url("**/dashboard") def test_elemento_visible(page: Page): page.goto("https://app.com") heading = page.locator("h1") expect(heading).to_be_visible() expect(heading).to_have_text("Bienvenido") def test_tabla_tiene_datos(page: Page): page.goto("https://app.com/reportes") filas = page.locator("table tbody tr") expect(filas).to_have_count(10) # exactamente 10 filas # O verificar que haya al menos 1: expect(filas.first()).to_be_visible()
Correr los tests
# Correr todos los tests $ pytest # Con más detalle $ pytest -v # Solo un archivo $ pytest test_login.py # Con browser visible (no headless) para debug $ pytest --headed # Generar reporte HTML $ pytest --html=reporte.html
expect() — assertions disponibles
to_have_title()El título de la página coincide
to_have_url()La URL actual coincide (acepta patterns con *)
to_be_visible()El elemento es visible en la pantalla
to_have_text()El texto del elemento coincide
to_have_count(n)El locator apunta exactamente a n elementos
to_be_enabled()El elemento no está disabled
to_have_value()El input tiene el valor esperado
not_to_be_visible()El elemento no es visible (o no existe)
pytest — Unit testing del parser (BeautifulSoup) Testear la lógica de extracción sin levantar un browser

import pytest from bs4 import BeautifulSoup # Función a testear def extraer_productos(html: str) -> list: soup = BeautifulSoup(html, "lxml") productos = [] for card in soup.select(".producto"): productos.append({ "nombre": card.select_one(".nombre").get_text(strip=True), "precio": card.select_one(".precio").get_text(strip=True) }) return productos # HTML de prueba (sin necesitar internet) HTML_EJEMPLO = """ <div class="producto"> <span class="nombre">Remera</span> <span class="precio">$4500</span> </div> <div class="producto"> <span class="nombre">Pantalón</span> <span class="precio">$8900</span> </div> """ def test_extrae_productos_correctamente(): resultado = extraer_productos(HTML_EJEMPLO) assert len(resultado) == 2 assert resultado[0]["nombre"] == "Remera" assert resultado[0]["precio"] == "$4500" def test_html_vacio_devuelve_lista_vacia(): resultado = extraer_productos("<html></html>") assert resultado == [] # Parametrize: testear múltiples casos en un solo test @pytest.mark.parametrize("precio_str,esperado", [ ("$4.500", 4500.0), ("$12.000,50", 12000.5), ("$100", 100.0), ]) def test_limpiar_precio(precio_str, esperado): limpio = float(precio_str.replace("$", "").replace(".", "").replace(",", ".")) assert limpio == esperado

@pytest.fixture — Setup y Teardown Reutilizar configuración entre tests (browser, driver, mocks)

Teardown: se destruye después.""" opts = Options() opts.add_argument("--headless") d = webdriver.Chrome(options=opts) d.implicitly_wait(5) yield d # ← el test recibe el driver aquí d.quit() # ← esto se ejecuta siempre al final (teardown) @pytest.fixture(scope="session") def base_url(): return "https://app.com" # En los tests, simplemente declarás el fixture como parámetro: def test_home_carga(driver, base_url): driver.get(base_url) assert "Bienvenido" in driver.page_source def test_nav_visible(driver, base_url): driver.get(base_url) nav = driver.find_element(By.TAG_NAME, "nav") assert nav.is_displayed()
scope= — ciclo de vida del fixture
"function"Se crea y destruye para cada test (default). Más aislamiento
"class"Compartido entre todos los tests de la misma clase
"module"Compartido entre todos los tests de un mismo archivo
"session"<
