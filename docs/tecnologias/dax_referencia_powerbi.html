Conceptos Fundamentales

Lo que tenés que entender antes de escribir tu primera medida DAX

fundamentos
Medida vs Columna Calculada La diferencia más importante de DAX
Característica Medida Columna Calculada
¿Cuándo se calcula? En tiempo de consulta (cada vez que el visual se renderiza) Al refrescar los datos (se guarda en el modelo)
Contexto de fila ❌ No tiene acceso directo a la fila ✅ Se evalúa fila por fila
Contexto de filtro ✅ Lo recibe del visual/segmentación ❌ No lo recibe
Consume memoria Mínima (no almacena resultados) Alta (una columna más en la tabla)
¿Para qué sirve? Métricas y KPIs: ventas totales, % margen, variaciones Clasificaciones, categorías, combinar columnas, flags
⚠️ El error más común: usar columnas calculadas para cosas que deberían ser medidas. Si el valor cambia según el filtro aplicado → usá medida.
Contexto de Filtro vs Contexto de Fila El corazón del motor de evaluación de DAX
Contexto de filtro — son todos los filtros activos cuando se evalúa una medida: segmentaciones, filtros de página, filtros de visual, jerarquías. Una medida SUM(Ventas[Monto]) suma distinto en cada celda de una matriz porque el contexto de filtro cambia por fila/columna.

Contexto de fila — existe al iterar una tabla (en columnas calculadas y en funciones iteradoras como SUMX). Permite acceder al valor de cada celda: Ventas[Precio] * Ventas[Cantidad].
Ejemplo — por qué esto importa
-- Contexto de filtro en una medida -- Si el usuario filtra por "Argentina", DAX solo suma las ventas de AR Total Ventas = SUM(Ventas[Monto]) -- Contexto de fila en columna calculada -- Se evalúa para CADA fila de la tabla Ventas Ventas[Subtotal] = Ventas[Precio] * Ventas[Cantidad]
💡 CALCULATE convierte el contexto de fila en contexto de filtro. Es por eso que es la función más poderosa de DAX.
Sintaxis DAX — Referencia rápida Cómo referenciar tablas, columnas y medidas
Sintaxis
-- Columna de tabla Ventas[Monto] -- Medida (corchetes sin tabla) [Total Ventas] -- Strings con comillas dobles IF(Clientes[Pais] = "Argentina", ...) -- Variables (muy recomendado) VAR _ventas = SUM(Ventas[Monto]) VAR _meta = SUM(Metas[Valor]) RETURN DIVIDE(_ventas, _meta, 0) -- Comentarios -- Una línea /* Múltiples líneas */
La convención _varName para variables locales es estándar en la comunidad DAX. Hace más fácil distinguirlas de columnas y medidas.
01

Medidas

Las funciones de agregación más utilizadas para KPIs y métricas

aggregation · kpi
SUM() · AVERAGE() · COUNT() · MIN() · MAX() Agregaciones básicas — las más usadas en cualquier modelo
DAX
Total Ventas = SUM(Ventas[Monto]) Ticket Promedio = AVERAGE(Ventas[Monto]) Cant Transacc = COUNT(Ventas[ID_Venta]) -- solo numéricos Cant Registros = COUNTA(Ventas[ID_Venta]) -- cualquier tipo Clientes Únicos = DISTINCTCOUNT(Ventas[ID_Cliente]) Precio Mínimo = MIN(Productos[Precio]) Precio Máximo = MAX(Productos[Precio])
Resultado (ejemplo con filtro "Argentina Q1")
Total Ventas → 1,250,000 Ticket Promedio → 4,350 Cant Transacc → 287 Clientes Únicos → 43
Diferencias clave
COUNTSolo cuenta valores numéricos. Ignora blancos y texto
COUNTACuenta cualquier valor no vacío (números, texto, fechas)
COUNTROWSCuenta filas de una tabla: COUNTROWS(Ventas). Más explícito
DISTINCTCOUNTCuenta valores únicos. Para métricas de cardinalidad
DIVIDE() División segura — nunca uses "/" para dividir medidas
DAX
-- ❌ Incorrecto: si denominador = 0 → error % Margen = [Margen Bruto] / [Total Ventas] -- ✅ Correcto: si denominador = 0 → devuelve 0 (o lo que definas) % Margen = DIVIDE([Margen Bruto], [Total Ventas], 0) -- Ejemplo real: % cumplimiento de meta % vs Meta = VAR _ventas = [Total Ventas] VAR _meta = SUM(Metas[Meta]) RETURN DIVIDE(_ventas, _meta, 0)
DIVIDE(numerador, denominador, [alternateResult])
numeradorExpresión numérica del dividendo
denominadorExpresión numérica del divisor
alternateResultQué devolver si denominador = 0 o BLANK. Default: BLANK()
IF() · SWITCH() Lógica condicional en medidas y columnas
DAX — IF
-- IF básico Estado Meta = IF([% vs Meta] >= 1, "✅ Cumplió", "❌ No cumplió") -- IF anidado (máximo 2-3 niveles, después usar SWITCH) Semáforo = IF([% vs Meta] >= 1, "Verde", IF([% vs Meta] >= 0.8, "Amarillo", "Rojo") )
DAX — SWITCH (más limpio que IF anidado)
-- SWITCH(TRUE()) → evalúa condiciones en orden Semáforo = SWITCH(TRUE(), [% vs Meta] >= 1, "🟢 Verde", [% vs Meta] >= 0.8, "🟡 Amarillo", [% vs Meta] >= 0.6, "🟠 Naranja", "🔴 Rojo" -- default ) -- SWITCH sobre valor exacto Nombre Region = SWITCH(Clientes[Cod_Pais], "AR", "Argentina", "BR", "Brasil", "CL", "Chile", "Otro" )
VAR / RETURN Variables locales — hacen el DAX legible y más rápido
VAR almacena un resultado intermedio. RETURN define qué devuelve la medida. Las variables mejoran la performance porque evitan que DAX evalúe la misma expresión varias veces.
DAX — sin VAR (repetitivo y lento)
Variación % = DIVIDE( SUM(Ventas[Monto]) - CALCULATE(SUM(Ventas[Monto]), SAMEPERIODLASTYEAR(Calendario[Fecha])), CALCULATE(SUM(Ventas[Monto]), SAMEPERIODLASTYEAR(Calendario[Fecha])), 0 )
DAX — con VAR (limpio y eficiente)
Variación % = VAR _actual = SUM(Ventas[Monto]) VAR _anio_ant = CALCULATE( SUM(Ventas[Monto]), SAMEPERIODLASTYEAR(Calendario[Fecha]) ) RETURN DIVIDE(_actual - _anio_ant, _anio_ant, 0)
💡 Usá VAR siempre que repitas una expresión más de una vez. También son clave para debugging: podés hacer RETURN _varIntermediate para inspeccionar valores.
BLANK() · ISBLANK() · COALESCE() Manejo de valores vacíos (el NULL de DAX)
DAX
-- BLANK() es el null de DAX -- Una medida devuelve BLANK cuando no hay datos en el filtro actual -- Detectar si una medida está en blanco Tiene Ventas = IF(ISBLANK([Total Ventas]), "Sin datos", "Con datos") -- COALESCE: devuelve el primer valor no-blank Precio Final = COALESCE(Productos[PrecioEsp], Productos[PrecioBase], 0) -- Forzar que una medida muestre 0 en vez de BLANK Ventas c/ceros = IF(ISBLANK([Total Ventas]), 0, [Total Ventas]) -- Forma más corta: Ventas c/ceros = [Total Ventas] + 0
⚠️ BLANK no es lo mismo que 0. En gráficos y tablas, BLANK omite la fila. A veces eso es lo que querés (no mostrar clientes sin ventas), a veces no.
02

Columnas Calculadas

Se evalúan fila por fila al refrescar. Ideales para clasificaciones y enriquecimiento del modelo.

row context · model
CONCATENATE() · FORMAT() · LEFT/RIGHT/MID() Manipulación de texto en columnas calculadas
DAX
-- Concatenar columnas (el & es más corto) Clientes[NombreCompleto] = Clientes[Nombre] & " " & Clientes[Apellido] -- Formatear números como texto Ventas[MontoFmt] = "$" & FORMAT(Ventas[Monto], "#,##0.00") -- Formatear fecha Ventas[MesAnio] = FORMAT(Ventas[Fecha], "MMMM YYYY") -- Extraer parte de un string Clientes[Iniciales] = LEFT(Clientes[Nombre], 1) Clientes[Sufijo] = RIGHT(Clientes[CUIT], 1) -- dígito verificador Clientes[CodPais] = MID(Clientes[Telefono], 2, 2) -- posición 2, 2 chars
Resultado
NombreCompleto → "Leonardo García" MesAnio → "Enero 2025" CodPais → "54"
FORMAT — códigos de formato comunes
"#,##0.00"Número con separador de miles y 2 decimales: 1,250.50
"0%"Porcentaje sin decimales: 85%
"0.0%"Porcentaje con 1 decimal: 85.3%
"DD/MM/YYYY"Fecha: 15/03/2025
"MMMM"Nombre del mes completo: Marzo
YEAR() · MONTH() · DAY() · WEEKDAY() · EOMONTH() Extraer partes de fechas (clave para tablas Calendario)
DAX — construir tabla Calendario completa
-- En la tabla Calendario (generada con CALENDARAUTO o CALENDAR) Calendario[Anio] = YEAR(Calendario[Fecha]) Calendario[Mes] = MONTH(Calendario[Fecha]) Calendario[NombreMes] = FORMAT(Calendario[Fecha], "MMMM") Calendario[Trimestre] = "Q" & QUARTER(Calendario[Fecha]) Calendario[Semana] = WEEKNUM(Calendario[Fecha], 2) Calendario[EsFinSemana] = IF(WEEKDAY(Calendario[Fecha], 2) > 5, 1, 0) Calendario[FinMes] = EOMONTH(Calendario[Fecha], 0)
WEEKDAY — parámetro return_type
return_type=1Domingo=1 ... Sábado=7
return_type=2Lunes=1 ... Domingo=7 (estándar europeo/LATAM)
EOMONTH(date, 0)Último día del mismo mes. EOMONTH(date, 1) = último del mes siguiente
RELATED() · RELATEDTABLE() Traer valores de tablas relacionadas (como un VLOOKUP en DAX)
DAX
-- RELATED: trae una columna de la tabla del lado "1" de la relación -- (equivale a VLOOKUP desde la tabla "muchos" hacia "uno") Ventas[CategoriaProducto] = RELATED(Productos[Categoria]) Ventas[NombreCliente] = RELATED(Clientes[RazonSocial]) -- RELATEDTABLE: desde el lado "1" hacia "muchos" — devuelve una tabla -- Se usa con funciones que aceptan tablas como COUNTROWS Clientes[CantCompras] = COUNTROWS(RELATEDTABLE(Ventas))
Resultado
-- En tabla Ventas: Fila 1: ID_Producto=5 → CategoriaProducto = "Electrónica" Fila 2: ID_Producto=5 → CategoriaProducto = "Electrónica" Fila 3: ID_Producto=9 → CategoriaProducto = "Ropa" -- En tabla Clientes: Spotify → CantCompras = 127 PPG → CantCompras = 43
⚠️ RELATED solo funciona en columnas calculadas (contexto de fila). En medidas, usá LOOKUPVALUE() o modelá la relación correctamente.
Columna de clasificación / segmento Agrupar en rangos o segmentos con SWITCH + TRUE()
DAX — segmentar clientes por monto de compra
Clientes[Segmento] = VAR _monto = SUMX(RELATEDTABLE(Ventas), Ventas[Monto]) RETURN SWITCH(TRUE(), _monto >= 500000, "A - Premium", _monto >= 100000, "B - Standard", _monto >= 10000, "C - Básico", _monto > 0, "D - Mínimo", "Sin compras" )
Resultado en la tabla Clientes
Spotify → "A - Premium" PPG → "B - Standard" Verizon → "A - Premium" NuevoX → "Sin compras"
03

CALCULATE y Modificadores de Filtro

La función más importante de DAX. Evalúa una expresión en un contexto de filtro modificado.

filter context · all · allexcept
CALCULATE() Evaluar una expresión cambiando el contexto de filtro activo
CALCULATE(expresión, filtro1, filtro2, ...)

Primero aplica los filtros (reemplazando o extendiendo el contexto actual), luego evalúa la expresión. Si no ponés filtros, convierte el contexto de fila en contexto de filtro.
DAX — ejemplos básicos
-- Ventas solo de Argentina (fija el filtro sin importar qué seleccionó el usuario) Ventas AR = CALCULATE(SUM(Ventas[Monto]), Clientes[Pais] = "Argentina") -- Ventas de múltiples países Ventas LATAM = CALCULATE( SUM(Ventas[Monto]), Clientes[Pais] IN {"Argentina", "Brasil", "Chile"} ) -- Ventas de productos con precio > 1000 Ventas Premium = CALCULATE( SUM(Ventas[Monto]), Productos[Precio] > 1000 )
ALL() · ALLEXCEPT() · ALLSELECTED() Remover filtros del contexto — esenciales para % del total
DAX
-- ALL(): elimina TODOS los filtros de una tabla o columna Total General = CALCULATE(SUM(Ventas[Monto]), ALL(Ventas)) -- % del total (el más usado) % del Total = VAR _actual = SUM(Ventas[Monto]) VAR _total = CALCULATE(SUM(Ventas[Monto]), ALL(Ventas)) RETURN DIVIDE(_actual, _total, 0) -- ALLEXCEPT(): elimina filtros de TODAS las columnas EXCEPTO las indicadas -- % dentro de cada categoría (mantiene el filtro de Categoria) % en Categoria = DIVIDE( SUM(Ventas[Monto]), CALCULATE(SUM(Ventas[Monto]), ALLEXCEPT(Productos, Productos[Categoria])), 0 ) -- ALLSELECTED(): respeta los filtros del usuario (segmentaciones) pero ignora el filtro del visual -- Útil para % que se actualiza según las selecciones del usuario % Seleccionado = DIVIDE( SUM(Ventas[Monto]), CALCULATE(SUM(Ventas[Monto]), ALLSELECTED(Ventas)), 0 )
Diferencias clave
ALL(Tabla)Ignora TODOS los filtros externos. El denominador es siempre el gran total
ALLEXCEPT(Tabla, Col)Ignora filtros de todas las columnas menos las indicadas
ALLSELECTED(Tabla)El total respeta segmentaciones pero no el filtro del visual. El más intuitivo para el usuario final
KEEPFILTERS() · REMOVEFILTERS() Control fino del comportamiento de los filtros en CALCULATE
DAX
-- Sin KEEPFILTERS: el filtro de CALCULATE REEMPLAZA al filtro del visual -- Si el usuario filtra "Electrónica" y vos ponés Pais="AR", el resultado -- es ventas de AR sin importar la categoría seleccionada. -- Con KEEPFILTERS: el filtro de CALCULATE se INTERSECTA con el filtro activo Ventas AR (respeta visual) = CALCULATE( SUM(Ventas[Monto]), KEEPFILTERS(Clientes[Pais] = "Argentina") ) -- REMOVEFILTERS: alias moderno de ALL() como argumento de CALCULATE Total sin filtros = CALCULATE( SUM(Ventas[Monto]), REMOVEFILTERS(Ventas) )
FILTER() Filtrar una tabla con condiciones complejas — devuelve una tabla
DAX
-- FILTER devuelve una tabla filtrada — se usa dentro de CALCULATE -- cuando necesitás condiciones más complejas que un simple = -- Ventas de clientes con más de 10 compras Ventas Frecuentes = CALCULATE( SUM(Ventas[Monto]), FILTER( Clientes, COUNTROWS(RELATEDTABLE(Ventas)) > 10 ) ) -- Ventas de los últimos 30 días Ventas 30d = CALCULATE( SUM(Ventas[Monto]), FILTER( Calendario, Calendario[Fecha] >= TODAY() - 30 ) )
⚠️ Preferí filtros simples col=valor dentro de CALCULATE cuando sea posible. FILTER itera la tabla completa, lo que puede ser lento en tablas grandes. Usalo solo cuando la condición lo requiera.
04

Time Intelligence

Comparaciones temporales — YTD, MTD, variación vs año anterior, períodos móviles.

fechas · ytd · yoy
⚙️ Requisito previo: para que funcionen todas las funciones de Time Intelligence necesitás una tabla Calendario marcada como "Tabla de fechas" en Power BI, con fechas continuas (sin días faltantes) que cubran todo el rango de datos. La forma más simple: Calendario = CALENDARAUTO()
TOTALYTD() · TOTALMTD() · TOTALQTD() Acumulados desde inicio de año / mes / trimestre
DAX
-- Ventas acumuladas desde el 1° de enero hasta la fecha del filtro Ventas YTD = TOTALYTD( SUM(Ventas[Monto]), Calendario[Fecha] ) -- Desde inicio de mes Ventas MTD = TOTALMTD(SUM(Ventas[Monto]), Calendario[Fecha]) -- Desde inicio de trimestre Ventas QTD = TOTALQTD(SUM(Ventas[Monto]), Calendario[Fecha]) -- YTD con año fiscal diferente (ej: año fiscal termina el 30/6) Ventas YTD Fiscal = TOTALYTD( SUM(Ventas[Monto]), Calendario[Fecha], "06/30" -- fin del año fiscal )
Resultado — si el usuario filtra Marzo 2025
Ventas MTD → suma de 1 al 31 de Marzo 2025 Ventas QTD → suma de Enero a Marzo 2025 (Q1) Ventas YTD → suma de Enero a Marzo 2025
SAMEPERIODLASTYEAR() · variación YoY Comparar el período actual con el mismo período del año anterior
DAX
-- Ventas del mismo período del año pasado Ventas Año Ant = CALCULATE( SUM(Ventas[Monto]), SAMEPERIODLASTYEAR(Calendario[Fecha]) ) -- Variación absoluta Δ vs Año Ant = [Total Ventas] - [Ventas Año Ant] -- Variación porcentual (YoY%) Var% YoY = VAR _actual = [Total Ventas] VAR _anterior = [Ventas Año Ant] RETURN DIVIDE(_actual - _anterior, _anterior, BLANK())
Resultado en un gráfico de barras por mes
Ene 2025: Ventas=120k | Ventas Año Ant=95k | Var% YoY=+26.3% Feb 2025: Ventas= 98k | Ventas Año Ant=88k | Var% YoY=+11.4% Mar 2025: Ventas=145k | Ventas Año Ant=BLANK| Var% YoY=BLANK ← no hay datos de Mar 2024
DATEADD() · PARALLELPERIOD() · DATESBETWEEN() Navegar entre períodos con más control que SAMEPERIODLASTYEAR
DAX
-- DATEADD: desplazarse N períodos en el tiempo Ventas Mes Ant = CALCULATE( SUM(Ventas[Monto]), DATEADD(Calendario[Fecha], -1, MONTH) ) Ventas 3 Meses Ant = CALCULATE( SUM(Ventas[Monto]), DATEADD(Calendario[Fecha], -3, MONTH) ) -- DATESBETWEEN: rango explícito de fechas Ventas H1 2025 = CALCULATE( SUM(Ventas[Monto]), DATESBETWEEN(Calendario[Fecha], DATE(2025,1,1), DATE(2025,6,30)) )
DATEADD — interval
DAYDesplaza N días
MONTHDesplaza N meses
QUARTERDesplaza N trimestres
YEARDesplaza N años (igual que SAMEPERIODLASTYEAR con -1)

Funciones Iteradoras (X)

Itera fila por fila una tabla, evalúa una expresión en cada fila y agrega los resultados.

sumx · averagex · rankx
SUMX() · AVERAGEX() · MINX() · MAXX() · COUNTX() Calcular una expresión por fila y luego agregar el resultado
DAX
-- SUMX: para cada fila de Ventas, calcula Precio*Cantidad y suma todo -- Necesario cuando no hay columna Subtotal precalculada Ingresos = SUMX(Ventas, Ventas[Precio] * Ventas[Cantidad]) -- Con descuento Ingresos Neto = SUMX( Ventas, Ventas[Precio] * Ventas[Cantidad] * (1 - Ventas[Descuento]) ) -- AVERAGEX: promedio ponderado (ticket promedio real) Ticket Prom Pond = AVERAGEX( VALUES(Ventas[ID_Venta]), CALCULATE(SUM(Ventas[Monto])) )
Comparación SUM vs SUMX
-- Si tenés columna Subtotal ya calculada: SUM(Ventas[Subtotal]) → más rápido, usa columna precalculada -- Si solo tenés Precio y Cantidad por separado: SUMX(Ventas, Ventas[Precio] * Ventas[Cantidad]) → necesario
RANKX() Ranking de elementos dentro de un contexto (Top N, posición)
DAX
-- Ranking de clientes por ventas (1 = mayor venta) Ranking Cliente = RANKX( ALL(Clientes), -- tabla donde rankear (ALL ignora filtros) [Total Ventas], -- expresión a evaluar para cada elemento , -- valor a rankear (si se omite, usa la expresión) DESC, -- DESC = mayor valor = rank 1 Dense -- Dense = sin saltos. Skip = salta si hay empates ) -- Mostrar solo el Top 5 Es Top 5 = IF([Ranking Cliente] <= 5, "Top 5", "Resto")
RANKX(tabla, expr, [val], [orden], [ties])
tablaSobre qué conjunto rankear. Usar ALL() para rankear globalmente
expresiónQué valor comparar para determinar el ranking
orden DESC/ASCDESC: mayor=1. ASC: menor=1 (para rankings de costo, errores, etc.)
Dense / SkipDense: sin saltos (1,2,2,3). Skip: con saltos si hay empate (1,2,2,4)
TOPN() Devuelve las N filas con el valor más alto/bajo de una expresión
DAX
-- Ventas de los 3 mejores clientes Ventas Top 3 = CALCULATE( SUM(Ventas[Monto]), TOPN(3, ALL(Clientes), [Total Ventas], DESC) ) -- Usar con SUMX para sumar algo del Top N Suma Top 5 Prods = SUMX( TOPN(5, ALL(Productos), [Total Ventas], DESC), [Total Ventas] )
06

Funciones de Tabla

Retornan tablas. Se usan dentro de otras funciones o para crear tablas calculadas en el modelo.

values · distinct · summarize
VALUES() · DISTINCT() · HASONEVALUE() Obtener los valores únicos de una columna respetando el contexto de filtro
DAX
-- VALUES: valores únicos + fila de BLANK si hay relaciones con blancos -- Se usa para iterar valores únicos Clientes Con Ventas = COUNTROWS(VALUES(Ventas[ID_Cliente])) -- HASONEVALUE: devuelve TRUE si el filtro dejó exactamente UN valor -- Útil para mostrar el valor cuando hay uno solo, o un mensaje si hay varios Pais Seleccionado = IF( HASONEVALUE(Clientes[Pais]), VALUES(Clientes[Pais]), "Múltiples países" ) -- SELECTEDVALUE: shortcut de HASONEVALUE + VALUES Pais = SELECTEDVALUE(Clientes[Pais], "Todos")
SUMMARIZE() · ADDCOLUMNS() Agrupar y agregar datos — el GROUP BY de DAX
DAX — tabla calculada
-- Resumen de ventas por País y Categoría Resumen = SUMMARIZE( Ventas, Clientes[Pais], -- columna de agrupación 1 Productos[Categoria], -- columna de agrupación 2 "Total", SUM(Ventas[Monto]) ) -- ADDCOLUMNS: agrega columnas a una tabla existente -- Patrón recomendado: SUMMARIZE para agrupar + ADDCOLUMNS para métricas Resumen2 = ADDCOLUMNS( SUMMARIZE(Ventas, Clientes[Pais]), "Ventas", [Total Ventas], "Clientes", [Clientes Únicos] )
💡 La combinación ADDCOLUMNS + SUMMARIZE es el patrón más correcto y performante para construir tablas de resumen en DAX. Evitá poner medidas directamente dentro de SUMMARIZE.
LOOKUPVALUE() Buscar un valor en una tabla sin necesitar relación — el VLOOKUP de DAX
DAX — columna calculada
-- LOOKUPVALUE(col_a_devolver, col_a_buscar, valor_buscado) Ventas[NombreProducto] = LOOKUPVALUE( Productos[Nombre], -- qué columna devolver Productos[ID_Producto], -- columna donde buscar Ventas[ID_Producto], -- valor a buscar "Sin nombre" -- valor si no encuentra (opcional) ) -- Con múltiples condiciones (como VLOOKUP con clave compuesta) Precios[PrecioEspecial] = LOOKUPVALUE( ListaPrecios[Precio], ListaPrecios[ID_Producto], Precios[ID_Producto], ListaPrecios[Pais], Precios[Pais], 0 )
💡 Si podés modelar una relación entre las tablas, preferí RELATED() — es más rápido. LOOKUPVALUE es para cuando no hay relación posible.
CALENDAR() · CALENDARAUTO() · GENERATE() Crear tablas calculadas de fechas y combinaciones
DAX — tablas calculadas
-- CALENDARAUTO: detecta el rango de fechas de todo el modelo Calendario = CALENDARAUTO() -- CALENDAR: rango manual Calendario = CALENDAR(DATE(2020,1,1), DATE(2025,12,31)) -- GENERATE: producto cartesiano de dos tablas (todas las combinaciones) -- Útil para tablas de escenarios o matrices de simulación Escenarios = GENERATE( VALUES(Regiones[Region]), VALUES(Productos[Categoria]) )
