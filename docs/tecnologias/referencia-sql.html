SELECTs y filtrado

-- Estructura básica
SELECT
    c.id_cliente,
    c.razon_social,
    c.cuit,
    COUNT(v.id_venta)   AS cant_ventas,
    SUM(v.monto)        AS total_facturado
FROM   clientes c
JOIN   ventas   v ON v.id_cliente = c.id_cliente
WHERE  c.activo = 1
  AND  v.fecha  BETWEEN '2025-01-01' AND '2025-12-31'
GROUP BY c.id_cliente, c.razon_social, c.cuit
HAVING  SUM(v.monto) > 100000
ORDER BY total_facturado DESC
LIMIT 20;

-- LEFT JOIN clásico: todos los clientes, hayan comprado o no
SELECT c.razon_social, COALESCE(SUM(v.monto), 0) AS total
FROM      clientes c
LEFT JOIN ventas   v ON v.id_cliente = c.id_cliente
GROUP BY  c.razon_social;

-- SELF JOIN: empleado y su gerente
SELECT e.nombre AS empleado, g.nombre AS gerente
FROM       empleados e
LEFT JOIN  empleados g ON g.id = e.id_gerente;

Funciones de ventana (Window Functions)

Una de las herramientas más poderosas de SQL moderno. Calculan valores sobre un conjunto de filas relacionadas sin colapsar el resultado como GROUP BY.

-- ROW_NUMBER: numeración por partición
SELECT
    id_cliente,
    fecha,
    monto,
    ROW_NUMBER() OVER (
        PARTITION BY id_cliente
        ORDER BY fecha DESC
    ) AS nro_compra
FROM ventas;

-- RANK vs DENSE_RANK
RANK()       OVER (ORDER BY monto DESC)  -- 1,2,2,4  (salta el 3)
DENSE_RANK() OVER (ORDER BY monto DESC)  -- 1,2,2,3  (sin saltos)

-- LAG / LEAD: valor de fila anterior / siguiente
SELECT
    fecha,
    monto,
    LAG(monto, 1) OVER (ORDER BY fecha) AS monto_mes_ant,
    monto - LAG(monto, 1) OVER (ORDER BY fecha) AS variacion
FROM ventas_mensuales;

-- SUM acumulado (running total)
SUM(monto) OVER (
    PARTITION BY id_cliente
    ORDER BY fecha
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
) AS acumulado

CTEs (Common Table Expressions)

Preferir CTEs sobre subconsultas anidadas — son más legibles y reutilizables.

-- CTE simple
WITH clientes_activos AS (
    SELECT id_cliente, razon_social, cuit
    FROM   clientes
    WHERE  activo = 1
),
ventas_2025 AS (
    SELECT id_cliente, SUM(monto) AS total
    FROM   ventas
    WHERE  YEAR(fecha) = 2025
    GROUP BY id_cliente
)
SELECT
    ca.razon_social,
    ca.cuit,
    COALESCE(v.total, 0) AS total_2025
FROM      clientes_activos ca
LEFT JOIN ventas_2025      v  ON v.id_cliente = ca.id_cliente
ORDER BY total_2025 DESC;

Optimización y buenas prácticas

✅ Usar índices correctamente
Crear índices en columnas usadas en WHERE, JOIN y ORDER BY. No indexar columnas con baja cardinalidad (ej: activo 0/1).
✅ SELECT específico
Nunca usar SELECT * en producción. Listar solo las columnas necesarias reduce I/O y mejora legibilidad.
✅ Filtrar temprano
Aplicar WHERE antes del JOIN cuando sea posible. Trabajar con el menor dataset posible en cada paso.
✅ Evitar funciones en WHERE
WHERE YEAR(fecha) = 2025 no usa índices. Usar WHERE fecha BETWEEN '2025-01-01' AND '2025-12-31'.
💡 Antes de optimizar, usá EXPLAIN / EXPLAIN ANALYZE para entender el plan de ejecución real.
⚠️ Un DELETE sin WHERE borra toda la tabla. Siempre verificar la condición con un SELECT primero.
